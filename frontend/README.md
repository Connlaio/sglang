# SGLang Frontend Chat Interface

这是一个基于 OpenAI 兼容 API 的 Web 前端聊天界面，支持与 SGLang 后端服务进行多轮对话。

## 功能特性

- 🗣️ **多轮对话**: 支持与 LLM 进行连续的多轮对话
- 📝 **对话历史管理**: 自动保存和管理对话历史，支持切换不同对话
- 🌊 **流式输出**: 支持实时流式输出，提供更好的用户体验
- ⚙️ **参数配置**: 可调节温度、最大长度等生成参数
- 📱 **响应式设计**: 支持桌面和移动设备
- 🎨 **美观界面**: 现代化的 UI 设计，简洁易用

## 使用方法

### 1. 启动 SGLang 服务器

首先需要启动 SGLang 服务器：

```bash
# 使用本地模型启动服务器
python -m sglang.launch_server --model-path meta-llama/Llama-3.1-8B-Instruct --port 30000

# 或者使用其他模型
python -m sglang.launch_server --model-path your-model-path --port 30000
```

### 2. 访问 Web 界面

服务器启动后，可以通过以下方式访问 Web 界面：

- 直接访问根路径: `http://127.0.0.1:30000/`
- 直接访问前端: `http://127.0.0.1:30000/frontend/index.html`

### 3. 配置连接

在界面顶部可以配置：
- **服务器地址**: SGLang 服务器的地址 (默认: http://127.0.0.1:30000)
- **模型选择**: 会自动检测当前服务器加载的模型

### 4. 开始对话

1. 在底部输入框中输入您的消息
2. 按 Enter 或点击"发送"按钮发送消息
3. AI 会根据您的输入生成回复

## 界面说明

### 主要区域

- **顶部配置区**: 服务器连接配置和模型选择
- **左侧边栏**: 对话历史列表，可以创建新对话或切换历史对话
- **中央聊天区**: 显示当前对话的消息历史
- **底部输入区**: 消息输入和参数配置

### 功能按钮

- **新对话**: 创建一个新的对话会话
- **刷新**: 重新获取服务器模型信息
- **发送**: 发送当前输入的消息
- **停止**: 中断正在生成的回复

### 参数设置

- **流式输出**: 开启后会实时显示生成过程
- **温度**: 控制生成文本的随机性 (0-2)
- **最大长度**: 限制单次回复的最大 token 数量

## 数据存储

- 对话历史会自动保存在浏览器的本地存储中
- 关闭浏览器或刷新页面后对话历史会保持
- 如需清除历史，可以清除浏览器的本地存储数据

## 技术特性

- **OpenAI 兼容**: 使用标准的 OpenAI API 格式与后端通信
- **无需安装**: 基于 Web 技术，无需额外安装客户端
- **实时通信**: 支持 Server-Sent Events (SSE) 流式响应
- **错误处理**: 完善的错误提示和连接状态显示

## 故障排除

### 连接问题

如果显示"连接失败"：
1. 检查 SGLang 服务器是否正常运行
2. 确认服务器地址配置正确
3. 检查网络连接和防火墙设置

### 模型加载问题

如果模型下拉框为空：
1. 确保服务器已完成模型加载
2. 点击"刷新"按钮重新获取模型信息
3. 检查服务器日志是否有错误信息

### 消息发送问题

如果消息无法发送：
1. 检查输入框是否有内容
2. 确认服务器连接状态正常
3. 查看浏览器控制台是否有错误信息

- `index.html`: 主页面结构
- `styles.css`: 样式文件
